WHAT IS COURSESENTRY?

CourseSentry is a self-hosted, real-time participant tracking and race management platform built for trail running, ultramarathon, cycling, and backcountry events where commercial cellular coverage is unreliable or absent. It unifies GPS position data from Meshtastic LoRa mesh radios, APRS amateur radio, Garmin InReach satellite communicators, and manual check-ins into a single live operational picture — giving race directors, net control operators, and the public the same situational awareness at the same time. Whether you are managing a fully-instrumented 100-mile ultra with 200 GPS trackers or a small local 50K where volunteers radio in bib numbers from aid stations, CourseSentry adapts to your resources and keeps everyone on the same page.

SETTING UP A NEW RACE

Race setup happens entirely in the Admin page. Work through the tabs in roughly this order before race day.

Step 1 — Create the Race

  1. Open Admin → Races and click NEW RACE.
  2. Enter the race name, date, and select a race format (see Race Types section below).
  3. Set units (US miles or metric), time format (12 h / 24 h), and speed display (pace or speed).
  4. Save. The race is created with status Upcoming.
  5. Click the race row to select it — a configuration bar appears with sub-tabs for the remaining steps.

Tip: If you ran a similar event before, use CLONE on an existing race to copy all settings, heats, classes, stations, and personnel. Then just adjust what changed.

Step 2 — Upload the Course

  1. Go to Admin → Courses and upload your GPX or KML track file. Supported sources: Strava export, Gaia GPS, CalTopo, Garmin Connect, Google Earth.
  2. If the file contains multiple tracks or routes, select the correct path from the dropdown.
  3. Optionally click EDIT MAP to fine-tune the track in the browser before saving.
  4. Back in Races → (your race) → COURSE, assign the uploaded course file to the race.
  5. Click AUTO-CREATE START/FINISH to seed station markers at the course endpoints.

Step 3 — Configure Stations

  1. Go to Races → STATIONS. The start and finish markers from Step 2 should already be there.
  2. Add any aid stations, checkpoints, turnarounds, or net-control stations your event uses.
  3. For each station, enter a name, latitude/longitude (or drag the pin on the map), and a type. See the Station Types reference for what each type does.
  4. Optionally set a cutoff time in minutes from race start for each aid station.

Step 4 — Set Up Heats (Optional)

  1. Go to Races → HEATS and click NEW HEAT.
  2. Name the heat (e.g., "Open", "Masters", "Wave 1"), assign a color and marker shape.
  3. Optionally set a per-heat start time if this wave starts separately from the race start.
  4. Repeat for each wave or category that needs its own color coding on the map.

Step 5 — Add Participants

  1. Go to Races → PARTICIPANTS. Add participants individually or bulk-import via IMPORT CSV.
  2. CSV columns: bib, name, tracker_id, heat, class, phone, emergency_contact, age, inreach_url.
  3. Assign a tracker ID to each participant who will carry a GPS-enabled device. For Meshtastic: use the node's long name or 8-character hex ID (e.g., !a4b92e3f). For APRS: use the operator's callsign (e.g., W1AW-5).
  4. Participants without a tracker ID can still be tracked manually via aid station check-ins.

Tracker ID matching: The system looks up incoming positions by tracker ID. If an APRS or Meshtastic node transmits and its ID matches a participant's tracker_id field, the position is attributed to that participant automatically. Unmatched nodes appear in the Infrastructure tab as discovered nodes — you can assign them from there.

Step 6 — Configure Tracker Infrastructure

See the Tracker Infrastructure section for full details. At minimum, enable the datasources your event uses (MQTT for Meshtastic, APRS for radio, etc.) and confirm the green status lights appear in the topbar.

Step 7 — Add Personnel (Optional)

  1. Go to Races → PERSONNEL and add volunteer names with optional tracker IDs and station assignments.
  2. Personnel with trackers appear on the operator map as distinct icons so you can see where your staff are relative to the course.

Step 8 — Enable Optional Features

In Races → (edit race settings), review the Features toggles:

FeatureWhat it does
Missing detectionFlags participants with no position update after a configurable interval (default 60 min)
Auto-log arrivalsCreates an arrival event automatically when a GPS-tracked participant enters a station's geofence
Auto-start on entryMarks participant as started when a manual check-in entry is created for them
Off-course alertsAlerts when a participant's GPS position is more than the configured distance from the course route
Stopped detectionAlerts when a participant has not moved for more than the configured interval (default 10 min)

Step 9 — Activate the Race

Required before activation: A course file must be assigned. The system will not allow activation without one.

  1. In Races, click ACTIVATE on your race row. Status changes to Active.
  2. Only one race can be active at a time (or more if your server is configured for multi-race). Activating a new race deactivates the current one.
  3. The operator and viewer pages now show live data for this race.

RACE TYPES & FORMATS

CourseSentry supports three race formats that affect how course progress, leaderboard position, and station logic work.

Point-to-Point

Participants start at one end of the course and finish at the other. Progress is calculated as distance traveled from the start station toward the finish, expressed as a percentage of total course distance. The leaderboard ranks participants by their percentage complete (highest first).

Station setup: Create a Start and a Finish station. Aid stations between them are ordered automatically by their position along the route.

Use for: Trail runs, road races, point-to-point cycling events.

Out-and-Back (OAB)

Participants travel from the start to a turnaround point and return to the finish. Progress is calculated as follows: 0–50% represents the outbound leg, 50–100% represents the return leg. Participants who have reached the turnaround but carry no GPS tracker are assumed at 50% until their next check-in. The leaderboard correctly ranks a returning participant above an outbound participant at the same raw distance from start.

Station setup: Use a Start/Finish (combined) station at the starting line and a Turnaround station at the midpoint. Aid stations on the course appear on both legs.

Use for: Out-and-back trail runs, 50K or 100K ultras with a single turnaround.

Fully Tracked vs. Manual Check-In

These are not separate race formats but rather modes of operation that can be mixed freely within any format.

ModeHow it worksBest for
Fully Tracked Every participant carries a GPS tracker (Meshtastic, APRS, or InReach). Position updates arrive automatically. Progress, pace, and alerts are computed continuously. Well-resourced events with tracker inventory, or small events where every participant has their own device.
Manual Check-In No GPS trackers. Aid station volunteers radio or call bib numbers to net control, who logs arrivals and departures in the operator interface. Position on the map is the last station checked in at. Low-budget events, large events with limited tracker inventory, or as a fallback for any participant whose tracker fails.
Hybrid Some participants carry GPS trackers; others are tracked manually. Both appear on the same map and leaderboard. Manually-tracked participants show at their last known station with reduced marker opacity. Most real events. Assign trackers to lead runners or participants in remote sections; track the rest manually.

COURSE & STATIONS

Uploading a Course File

Go to Admin → Courses and drag or browse to a GPX or KML file. CourseSentry extracts track points (the route line) and named waypoints (which become candidate aid station locations). Files up to 50 MB are supported; typical course files are well under 1 MB.

If the file has multiple tracks or route segments, a path selector appears — choose the one that matches your course.

Editing the Course

Select a course in the list and click EDIT MAP (only available when the course is not assigned to an active race). The map editor opens in a new tab with:

  • Edit Vertices (pencil toolbar button) — drag any point on the track. Click the midpoint of a segment to insert a new vertex. Right-click a vertex to remove it.
  • + WAYPOINT button — click anywhere on the map to drop a named waypoint.
  • Drag any waypoint marker to reposition it. Click a waypoint marker to rename or delete it.
  • SAVE — writes the edited geometry back to the server as a GPX file.

Station Ordering

When a course is assigned to a race, stations are automatically ordered along the route by projecting each station's coordinates onto the course line. You do not need to manually sort them — the system determines which station comes first, second, etc., and uses that order for the leaderboard and progress calculations.

Importing Waypoints as Stations

Named waypoints embedded in your GPX or KML file (e.g., "Aid 1", "Turnaround") appear in the course preview. You can bulk-import these as stations by clicking IMPORT WAYPOINTS in the Course tab of the race configuration. Each waypoint becomes an aid-type station that you can then edit individually.

Heats

Heats are colored start waves shown as distinct marker types on the map. Configure them under Races → HEATS. Each heat gets a color and one of six shapes: circle, triangle, square, diamond, star, or pentagon. When a participant has a heat assigned, their map marker uses that color and shape — making it easy to visually distinguish runners by wave at a glance.

If a heat has its own start time, all participants in that heat begin timing from that moment rather than the race start. This is important for staggered starts — without per-heat start times, everyone's pace and ETA calculations use the same reference clock.

CSV Import

Prepare a CSV with column headers. Required: bib, name. Optional: tracker_id, heat (heat name), class (class name), phone, emergency_contact, age, inreach_url. Heat and class values are matched by name — create them in the HEATS and CLASSES tabs first. Unknown heat or class names are silently skipped.

TRACKER INFRASTRUCTURE

CourseSentry can receive position data from four sources simultaneously. Enable each one under the race's RF settings.

Meshtastic (MQTT)

Meshtastic is a LoRa-based open-source mesh radio network. Devices communicate over unlicensed spectrum (915 MHz in North America) and relay positions via a mesh — meaning participants do not need direct line-of-sight to a gateway, only to any other node in the mesh.

Setup

  1. Configure each Meshtastic device with your MQTT server, channel name, and channel key (PSK). Devices must be in the same channel to communicate.
  2. In CourseSentry, go to the race's RF settings and enter the same MQTT host, port, channel, and PSK. The default server (apps.k7swi.org) works for testing but you should run your own broker for production events.
  3. Enable MQTT in the datasource settings. The MQTT status light in the topbar turns green when connected.
  4. Assign each participant the long name or hex node ID (e.g., !a4b92e3f) of their device in the participant's tracker_id field.

MQTT RF Tech: The default mode is Meshtastic-formatted JSON position packets. If your mesh gateway publishes raw APRS-format packets over MQTT, select the APRS-over-MQTT mode instead.

APRS (HF/VHF/UHF / TCP/AX.25/LoRa)

APRS (Automatic Packet Reporting System) transmits position packets over amateur radio across a wide range of bands and transports — HF, VHF, UHF, TCP/IP (APRS-IS), AX.25 packet radio, and LoRa. CourseSentry connects to the APRS-IS internet backbone to receive these packets in real time. A valid amateur radio license is required to transmit APRS; receiving requires no license.

Setup

  1. Enable APRS in the race's RF settings and enter the race's tactical callsign (1–6 alphanumeric characters, e.g., RACE1 or your own callsign).
  2. Assign each APRS participant's amateur callsign to their tracker_id field (e.g., W1AW-5).
  3. The APRS status light turns green when connected to APRS-IS. Positions begin arriving automatically as beacons are received.

License requirement: Transmitting on APRS requires a valid FCC amateur radio license (Technician class or higher). CourseSentry's APRS integration is receive-only by default; two-way messaging via APRS requires a licensed control operator managing the gateway.

KISS TNC (Serial Radio)

A KISS TNC (Terminal Node Controller) connects CourseSentry to a traditional VHF/UHF radio for direct APRS packet transmission and reception — without needing internet connectivity. This is the recommended setup for events in areas with no cell or internet coverage.

Setup

  1. Enable TNC in the race settings and configure the tactical callsign and RF digipeater path (typically WIDE1-1).
  2. Connect a TNC device to the computer running CourseSentry via WebSerial (Chrome/Edge over HTTPS).
  3. The TNC light in the topbar shows how many clients are connected and which one is the primary transmitter (TX).

Multi-client TNC: Multiple computers can connect to the same TNC session. Only the "primary" client transmits; others receive. This lets a backup laptop monitor traffic without double-transmitting.

Garmin InReach (Satellite)

Garmin InReach devices communicate over the Iridium satellite network — they work anywhere on Earth with no ground infrastructure. Coverage comes at the cost of update frequency (typically every 10 minutes) and requires a paid Garmin subscription per device.

Setup

  1. Each participant with an InReach device must share their tracking link from the Garmin Explore portal (or the InReach app). The link looks like https://share.garmin.com/CALLSIGN.
  2. Paste this URL into the participant's InReach URL field. CourseSentry polls the link periodically and extracts the last reported position.
  3. No race-level toggle is needed — InReach polling activates automatically for any participant who has an InReach URL configured.

PUBLIC VIEWER

The public viewer is a read-only, token-gated page you can share with spectators, family members, crews, and the media. It shows the live map and/or leaderboard with no login required.

Generating a Viewer Link

  1. In Admin → Races, select your race and open its settings.
  2. Click GEN VIEWER to generate a unique access token. A shareable URL appears.
  3. Share the URL via social media, email, or embed it in your race website.
  4. To invalidate the link (e.g., after the race), click REVOKE and the token is immediately disabled.

BEFORE THE RACE

The Operator page is the real-time race command interface, typically running on the net control station's laptop or tablet. Open it from the Admin topbar (OPERATOR VIEW) or navigate directly to the race-select page.

Pre-Race Checklist

  • Confirm the correct race is active (race name shown in the topbar pill).
  • Verify datasource status lights are green for the sources you are using (MQTT, APRS, TNC, InReach).
  • Confirm participant trackers appear in the Infrastructure tab — nodes broadcasting positions show up there as discovered.
  • Check the map: the course route (orange line) and all station markers should be visible. If not, verify the course is assigned in Admin → Races → COURSE.
  • If using offline maps, confirm the offline tiles status shows READY before going into the field.
  • Test two-way messaging if using Meshtastic or APRS — send a test message to a known node.

Setup Warning Bar

If any critical configuration is missing (no course assigned, no stations, no tracker infrastructure enabled), a yellow warning bar appears at the top of the operator page describing what needs attention. Address all warnings before race day.

STARTING THE RACE

Start Window (Optional)

For events with a defined pre-race window (e.g., participants can start anytime between 6:00 AM and 6:30 AM), use the Start Window feature:

  1. Click OPEN START WINDOW in the operator topbar. The start window opens and a countdown timer begins.
  2. As participants cross the start line, check them in manually or let GPS auto-detection handle it (if auto-start is enabled and they have trackers).
  3. When the window closes (or you click to close it), any remaining participants who have not been checked in are marked DNS.

Starting the Race

  1. At race time, click START RACE in the operator topbar. This sets the official race start time to the current clock.
  2. All participants marked as Active have their start time recorded. The leaderboard begins calculating pace and ETA.
  3. For participants without GPS trackers, create a Start event manually (select the participant on the leaderboard, then log a start event) to begin their timing.

Manual Start Events

For fully manual or hybrid events, you can create individual start events as participants cross the line:

  1. Click a participant's row on the leaderboard to open their detail panel.
  2. Click LOG EVENT and select Start. Optionally adjust the timestamp if there was a delay in logging.
  3. The participant's status changes to Active and timing begins from the logged start time.

MANAGING THE RACE

Leaderboard

The leaderboard shows all participants sorted by race position (highest progress percentage first) by default. Use the sort buttons to switch to BIB order, pace, name, or heat. Click any row to see that participant's detail panel with full event history, last GPS position, battery level, and signal strength.

Manual Check-Ins (Aid Station Reporting)

When a volunteer at an aid station radios a bib number to net control, log it here:

  1. Find the participant by BIB on the leaderboard (use the search/filter if needed) or click CHECK-IN to open the batch check-in modal.
  2. Select the station and event type: ARRIVE (reached the station) or DEPART (left the station).
  3. Enter any notes (e.g., "moving well", "asked for medical").
  4. The participant's position on the map updates to the station location and their leaderboard row reflects the new progress.

Departure is the significant marker: Departure time drives leaderboard position, pace, and ETA calculations. Always log a DEPART event when a participant leaves a station. If Auto-log arrivals is enabled, arrival events are created automatically when a GPS tracker enters a station's geofence — this is a convenience bonus, but departure must still be logged.

Logging a DNF

  1. Click the participant's row on the leaderboard.
  2. Click LOG EVENT → DNF. Optionally note the station or location where they withdrew.
  3. The participant is moved to the DNF section of the leaderboard and their marker changes to indicate withdrawn status.

Map Controls

  • Base layer: Switch between Topo, Satellite, Street, and Dark in the topbar selector. If offline maps are enabled and ready, only Topo and Satellite are available.
  • Nametags: Toggle persistent name/BIB labels on all map markers.
  • Personnel: Toggle visibility of volunteer/staff markers.
  • Weather overlays: Available if a weather API key is configured. Select precipitation, wind, radar, lightning, or other layers. Adjust overlay opacity with the slider.

Aid Station Operator (Mobile View)

Volunteers at remote aid stations can use the mobile station interface (/mobileop) on a smartphone or tablet. They log in with a station-role account, select their station, and check in arriving participants by BIB. Check-ins from the field appear instantly in the operator view at net control.

ALERTS & INCIDENTS

When alert features are enabled, CourseSentry automatically monitors GPS-tracked participants and flags anomalies. The alert count badge in the topbar shows how many participants have active alerts. Alert rows on the leaderboard blink to draw attention.

Alert TypeTriggerCleared when
OFF-COURSE GPS position is more than the configured distance (default 100 m) from the course route Participant returns within the threshold distance
STOPPED No movement detected for longer than the configured interval (default 10 min) Movement is detected
MISSING No GPS position update received for longer than the missing timer (default 60 min) A new position packet is received

Responding to Alerts

  1. Click the blinking participant row to open their detail panel. Review their last known position, timestamp, and event history.
  2. If they have a Meshtastic or APRS tracker, send them a direct message from the Messaging panel to check on their status.
  3. Contact the nearest aid station or sweep crew to request a visual check.
  4. If the participant is confirmed safe and stationary (e.g., resting at a non-station location), the alert will clear automatically when they resume movement. You can also log a manual note.
  5. If a search and rescue response is initiated, log a DNF or custom note event to record the decision in the event log.

Alert thresholds: Default values are conservative starting points. Adjust the missing timer, stopped time, and off-course distance in race settings to match your event's terrain, expected pace, and normal aid station dwell times. A 100-miler with aid station stops may need a 90-minute missing threshold; a fast 10K needs 15 minutes.

MESSAGING

If the race has messaging enabled and at least one of MQTT (Meshtastic) or APRS/TNC is connected, you can send short text messages directly to participant trackers from the operator interface.

Sending a Message

  1. Open the Messages panel on the right side of the operator page.
  2. Select the target: a specific participant (their tracker will be addressed by node ID or callsign), or type a callsign/node ID directly.
  3. Type your message (maximum 67 characters) and click SEND.
  4. The message appears in the queue with status: QUEUED → SENT or ERROR.

Delivery vs. receipt: "Sent" means the packet was transmitted. APRS-IS and Meshtastic do not always provide delivery confirmation to end devices — especially over mesh hops. For critical communications, follow up with a voice radio call if the situation warrants it.

FINISHING THE RACE

  1. As participants finish, their arrival at the Finish station (GPS geofence or manual check-in) creates a Finish event and marks their status as Finished. Their finish time and pace are recorded.
  2. Any participants still on course at cutoff time should be logged as DNF with a note referencing the cutoff.
  3. When all participants are accounted for, click END RACE in the operator topbar. The race status changes to Past.
  4. The public viewer link remains valid until you revoke the token, so spectators can continue to view final results.

Results preservation: All events, positions, and times are stored in the database permanently. Past races are visible in the Admin races list and their data is retained for records.
